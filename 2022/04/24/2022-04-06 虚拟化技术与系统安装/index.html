虚拟化技术

虚拟化技术简述

通过虚拟机监控程序将电脑物理资源进行有效分隔（例如:CPU、硬盘等），并将这些分隔出来的不同、单独的环境创建为多个模拟环境或专用资源。

VM虚拟机

通过软件来模拟具有完整硬件系统功能的、运行在一个完全隔离的环境系统中的完整计算机，每个虚拟出来的计算机都有独立的CMOS、硬盘和操作系统

docker

docker是一个轻量化的虚拟技术、与主机共享操作系统内核、共享磁盘空间、无需为docker单独分配出硬件资源，可以类似于软件一样进行运行。

VM与docker比较

对比项目 docker容器 虚拟机（VM）
操作系统 与宿主机共享OS 宿主机OS上运行虚拟机OS
存储大小 镜像小，便于存储与传输 镜像庞大
运行性能 几乎无额外性能损失 操作系统额外的CPU、内存消耗
移植性 轻便、灵活，适应于linux 笨重、与虚拟化技术耦合度高
硬件亲和性 面向软件开发者 面向硬件运维
部署速度 快速，秒级 较慢，分钟级

WM虚拟机得三种连网方式

vmnet1是为host-only方式服务的，vmnet8是为NAT方式服务的
使用Host-Only模式的guest系统与主机通信需要VMnet1网卡的支持；
通过NAT方式上网的guest系统与主机通信需要VMnet8网卡的支持；
使用桥接模式上网需要网络中存在DHCP服务器，且提供服务。

桥接模式

类似于将物理机网卡虚拟为一台交换机，物理机通过网桥方式与虚拟机进行通信，同时物理机与虚拟机都接在这台交换机端口上。桥接模式下，IP地址需要与物理主机处在同一网段、网关与DNS需要与主机相同。

NAT地址转换

虚拟机通过虚拟NAT设备进行IP地址转换直接与物理主机网卡相连，实现联网。虚拟网卡8与物理主机连接实现虚拟机与物理主机之间的网络通信。

Host-only仅主机模式

虚拟机网卡与物理主机相连，实现虚拟机与物理主机之间局域网的连接。

VM的两种克隆模式

完整克隆模式

完全克隆父主机的状态，并不复制父主机的快照，完全克隆的虚拟机不与父主机共享磁盘，相当于是一个独立的虚拟机。

链接克隆模式

依赖于父主机，由父主机快照创建而成共享磁盘空间。父主机损坏或快照点删除将影响链接克隆主机使用。

VM快照

用于备份及恢复虚拟机某个时间段的状态。

kali linux 与windows server 2008安装

kali linux安装

1.输入网址https://www.kali.org/进入kali linux官网

2.点击下载进入kali下载页面

3.选择安装方式

4.将下载好kali linux的安装包进行解压至文件

5.通过WM虚拟机文件选项中打开kali linux所在文件夹

6.选中vm启动文件，进行导入

7.创建成功

windows server 2008 R2

1.输入系统镜像网址 https://hellowindows.cn/

2.选择需要安装的镜像

3.打开wm虚拟机
点击文件-新建虚拟机-典型安装

4.选择下载好的虚拟机镜像文件，点击下一步；

5.跳过密钥输入，选择是，进入下一步。


6.为虚拟机命名，并选择存放位置

7.选择虚拟机磁盘容量及磁盘安装方式，进入下一步

8.查看虚拟机配置信息，进入下一步

9.开启虚拟机，进入安装状态

10.安装完成
